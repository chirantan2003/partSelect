# Data — PartSelect Parts Assistant

This directory holds raw scraped data from PartSelect, used as the source of truth for seeding the PostgreSQL database. The files are generated by the scraper pipeline and consumed by the loader (`backend/scraper/load.py`).

---

## Directory Structure

```
data/
└── raw/
    ├── parts_raw.json             # Full part catalog (listing + detail pages)
    ├── parts_checkpoint.json      # Scraper progress checkpoint
    ├── repair_symptoms_raw.json   # Repair symptom guides + part mappings
    ├── blogs_raw.json             # Blog articles (repair how-tos, guides)
    └── blogs_checkpoint.json      # Blog scraper progress checkpoint
```

---

## File Descriptions

### `raw/parts_raw.json` (~3.5 MB)

Contains the full scraped part catalog. Each entry represents one part scraped from PartSelect's facet search listing pages and individual part detail pages.

**Schema per entry:**
```json
{
  "ps_number": "PS11752778",
  "mpn": "WPW10321304",
  "name": "Door Shelf Bin",
  "description": "...",
  "price_cents": 4740,
  "in_stock": true,
  "image_url": "https://...",
  "rating": 4.5,
  "review_count": 123,
  "appliance_type": "refrigerator",
  "video_url": "https://...",
  "compatible_models": ["WRS322FDAM00", "WRS325SDHZ00", ...],
  "symptoms": ["Door won't open or close", "Ice maker not making ice"],
  "installation_steps": [
    { "step_no": 1, "text": "...", "difficulty": "Easy", "est_minutes": 15 }
  ],
  "alt_numbers": ["WPW10321304", "W10321304"]
}
```

**How it's used:** `backend/scraper/load.py` reads this file and upserts records into:
- `parts` table
- `appliance_models` table (via `compatible_models`)
- `part_compatibility` table (the compatibility matrix)
- `part_cross_refs` table (via `alt_numbers`)
- `symptoms` + `part_symptoms` tables (via `symptoms` list)
- `installation_steps` table
- `related_parts` table

---

### `raw/parts_checkpoint.json` (~3.5 MB)

A scraper progress file. Tracks which part URLs have already been scraped so the scraper can resume after an interruption without re-fetching completed parts.

> **Do not delete** while a scrape is in progress. Safe to delete before a fresh full re-scrape.

---

### `raw/repair_symptoms_raw.json` (~89 KB)

Contains scraped data from PartSelect's repair symptom pages (e.g., `/Repair/Refrigerator/Noisy/`). Each entry maps a symptom to repair guide text and associated PS part numbers.

**Schema per entry:**
```json
{
  "appliance_type": "refrigerator",
  "symptom_slug": "Not-Making-Ice",
  "symptom_display": "Ice Maker Not Making Ice",
  "source_url": "https://www.partselect.com/Repair/Refrigerator/Not-Making-Ice/",
  "intro": "If your refrigerator ice maker is not making ice...",
  "steps": ["Check the water supply line...", "Inspect the water inlet valve..."],
  "part_ps_numbers": ["PS11752778", "PS11723157"],
  "video_url": "https://..."
}
```

**How it's used:** `load.py` inserts this data into `symptoms`, `repair_guides`, and `part_symptoms` tables. The `intro` and `steps` text is also chunked and embedded (768-dim via Gemini) into `repair_guides.embedding` for semantic search.

**Covered symptoms:**

| Refrigerator | Dishwasher |
|---|---|
| Noisy | Noisy |
| Leaking | Leaking |
| Will Not Start | Will Not Start |
| Ice Maker Not Making Ice | Door Latch Failure |
| Fridge Too Warm | Not Cleaning Dishes |
| Not Dispensing Water | Not Draining |
| Fridge and Freezer Too Warm | Won't Fill With Water |
| Door Sweating | Won't Dispense Detergent |
| Light Not Working | Not Drying Dishes |
| Fridge Too Cold | |
| Freezer Too Cold | |

### `raw/blogs_raw.json` (~870 KB)

Blog articles scraped from PartSelect's blog covering repair guides, how-to articles, error code explanations, and use-and-care tips.

**Schema per entry:**
```json
{
  "url": "https://www.partselect.com/blog/...",
  "title": "How to Fix a Leaking Refrigerator",
  "appliance_type": "refrigerator",
  "brand": "Whirlpool",
  "body": "...",
  "image_url": "https://...",
  "likely_part_ps_numbers": ["PS11752778"]
}
```

**How it's used:** `load.py` embeds the `body` text and inserts records into `repair_guides`. This is the primary source for semantic troubleshooting search (`search_repair_guides` tool).

**Blog topics scraped:** `repair`, `error-codes`, `how-to-guides`, `testing`, `use-and-care`

---

### `raw/blogs_checkpoint.json` (~762 KB)

Scraper progress file for blog articles. Same purpose as `parts_checkpoint.json` — enables resume on interruption.

## Regenerating the Data

To re-scrape all data from scratch:

```bash
# Delete checkpoints to force a full re-scrape (optional)
# del data\raw\parts_checkpoint.json
# del data\raw\blogs_checkpoint.json

# Run the full seeding pipeline
python -m backend.scraper.seed
```

The seed script:
1. Scrapes part listings + detail pages → `parts_raw.json`
2. Scrapes repair symptom pages → `repair_symptoms_raw.json`
3. Scrapes blog articles → `blogs_raw.json`
4. Generates embeddings for symptoms + repair guides
5. Upserts everything into PostgreSQL

> ⏱ Full scrape takes **several hours** due to polite rate limiting (2–4 second delays). Checkpoints allow safe resumption.

## 💬 Testing Data Coverage with Prompts

You can verify that the seeded database tables are functioning correctly by submitting these prompts in the chat:

### 1. Verifying Symptom Keyword Matching (`symptoms` table)
* **Prompt:** `My refrigerator is Leaking.`
* **Data Flow:** Triggers a keyword query:
  ```sql
  SELECT id FROM symptoms WHERE description ILIKE '%Leaking%' AND appliance_type = 'refrigerator';
  ```
* **Expected:** Matches the `"Leaking"` symptom row and returns related parts.

### 2. Verifying Vector Similarity Search (`symptoms` + `pgvector`)
* **Prompt:** `dishwasher water wont leave`
* **Data Flow:** Computes a 768-dimensional embedding vector of the prompt and queries the HNSW index:
  ```sql
  SELECT description, (embedding <=> $1) as distance 
  FROM symptoms 
  WHERE appliance_type = 'dishwasher' 
  ORDER BY distance LIMIT 1;
  ```
* **Expected:** Cosine similarity matches `"Not Draining"` (distance < 0.3) and yields drain pumps.

### 3. Verifying Compatibility Matrix (`part_compatibility` table)
* **Prompt:** `Is part number PS10065979 compatible with WDT780SAEM1?`
* **Data Flow:** Executes a deterministic check on the join table:
  ```sql
  SELECT pc.* 
  FROM part_compatibility pc
  JOIN parts p ON p.id = pc.part_id
  JOIN appliance_models m ON m.id = pc.model_id
  WHERE p.ps_number = 'PS10065979' AND m.model_number = 'WDT780SAEM1';
  ```
* **Expected:** Returns the link record, outputting a compatible verdict.
